# 老王翻译缓存配置指南

艹！老王给你详细说明一下缓存是怎么工作的，以及怎么配置！

## 🔥 缓存架构说明

### 当前缓存方案：
1. **内存缓存** (临时) - 在Cloudflare Workers内存中
2. **KV存储缓存** (持久) - 在Cloudflare KV中，可选配置

## 📍 缓存位置详解

### 1. 内存缓存 (默认启用)
- **位置**: Cloudflare Workers运行时内存
- **生命周期**: Workers实例重启后丢失
- **优点**: 访问速度极快
- **缺点**: 不持久，多实例不共享
- **容量**: 限制500条记录

### 2. KV存储缓存 (可选)
- **位置**: Cloudflare KV存储
- **生命周期**: 24小时自动过期
- **优点**: 持久化，全球共享
- **缺点**: 需要额外配置
- **容量**: 几乎无限制

## ⚙️ KV存储配置步骤

### 步骤1: 创建KV命名空间
```bash
# 在你的项目目录下运行
npx wrangler kv:namespace create "TRANSLATION_CACHE"
```

### 步骤2: 更新wrangler.toml
在你的`wrangler.toml`文件中添加：
```toml
[[kv_namespaces]]
binding = "TRANSLATION_CACHE"
id = "你的KV命名空间ID"  # 从步骤1获得
```

### 步骤3: 部署更新
```bash
npx wrangler deploy
```

## 🚀 缓存工作流程

```
用户请求翻译
    ↓
检查内存缓存 → 命中 → 立即返回
    ↓ 未命中
检查KV缓存 → 命中 → 返回并更新内存缓存
    ↓ 未命中
调用AI翻译
    ↓
保存到内存缓存
    ↓
保存到KV缓存 (24小时过期)
    ↓
返回结果
```

## 📊 缓存效果

### 没有KV存储时：
- ✅ 同一Workers实例内的重复请求会被缓存
- ❌ Workers重启后缓存丢失
- ❌ 不同地区的用户不共享缓存

### 配置KV存储后：
- ✅ 全球用户共享缓存
- ✅ 24小时内的重复翻译直接返回
- ✅ 大幅减少AI API调用
- ✅ 提高响应速度

## 🔧 环境变量说明

老王的代码会自动检测以下环境变量：
- `TRANSLATION_CACHE`: KV存储绑定名称

如果没有配置KV存储，系统会：
1. 只使用内存缓存
2. 在控制台输出提示信息
3. 正常工作，但缓存不持久

## 💰 成本说明

### KV存储费用 (Cloudflare)：
- **免费额度**: 每天100,000次读取，1,000次写入
- **超出费用**: 读取$0.50/百万次，写入$5.00/百万次
- **存储费用**: $0.50/GB/月

### 老王建议：
对于个人使用，免费额度完全够用！

## 🛠️ 故障排除

### 问题1: KV缓存不工作
```javascript
// 检查环境变量
console.log('KV绑定:', env.TRANSLATION_CACHE ? '已配置' : '未配置');
```

### 问题2: 缓存命中率低
- 检查缓存键是否正确生成
- 确认文本、语言、模型参数一致

### 问题3: 内存使用过高
- 当前限制500条内存缓存
- 可以调整`translationCache.size > 500`这个数值

## 📈 性能优化建议

1. **启用KV缓存** - 获得最佳一致性
2. **合理设置过期时间** - 当前24小时
3. **监控缓存命中率** - 通过日志查看
4. **定期清理** - KV会自动过期
